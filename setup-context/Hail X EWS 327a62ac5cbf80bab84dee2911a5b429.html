Hail WordPress+ Integration

  • Instructions

    With WordPress+, Hail users can push content directly to selected areas of a WordPress website using configured connections.

    Because WordPress is highly modular, every site setup will be slightly different, but the steps below cover the general configuration needed to make WordPress+ work smoothly with Hail.

    Overview

    With WordPress+, articles and publications from Hail can be pushed into WordPress as standard post objects.

    The following content is pushed automatically:

    • title
    • featured image
    • body content
    • author
    • publish date

    Once pushed, this content can be used in WordPress like any other post, including in archives, category pages, tag filters, widgets, and custom page layouts.

    Each connection can be configured to apply a single WordPress category to the posts it creates. If no category is assigned, posts will default to Uncategorised.

    Setup Guide

    WordPress

    1. Create Categories

    In WordPress, go to Posts → Categories and create the categories you want each connection to push into. Recommended connection categories include Articles, Publications, Notices, and Alerts.

    Hail will automatically push all public tags into WordPress, so any further filtering can usually be managed there.

    As an alternative, you can create categories based on page destinations instead of content types. This can provide a clearer UX when content is pushed from Hail. For example, you might create categories such as Homepage, Newsletter Page, or Sports Page, depending on how the site is structured.

    2. Create an Application Password

    Go to Users in WordPress and select an admin user.

    Hail

    Create a WordPress+ Connection

    1. Go to Hail and select the Organisation from the top-right menu. If you only have 1 - then it will be the default.
    1. Select Connections tab from the left side bar
    1. SelectBlogging tab and then Wordpess Plus connection
    1. Enter your Wordpress site URL (e.g. https://wordpress-site.co.nz)
    1. Enter your username
      Note: not the application password name, the actual WP users username
    1. Enter your Application Password
    1. Click Authorise
    At this point Hail will make the necessary calls to your Wordpress site, and the next steps allow you to customise your new Wordpress+ connection within Hail. After customising you will be able to push your Hail content into Wordpress from the standard connections panel inside Hail’s Article & Publication edit screens

    There is a form submission / validation issue if “Which post type should be applied to posts for this connection?” is left blank. NONE should be manually selected in this case

    When editing an Article or Publication in Hail, users will see the new Push to WordPress connection in the right-hand sidebar. Clicking this will send the content to WordPress as a post using the category assigned to that connection. In WordPress, you can then use blog modules or post queries filtered by category to display the pushed content.

    Gotchas

    Wordfence

    Troubleshooting: If Wordfence has disabled application passwords you will need to enable (uncheck) this in Wordfence config. 
    remove_action('request', 'wordfence::preventAuthorNScans');

    (Optional) Network & Plugin considerations:

    HTTP (and not HTTPS)

    • Add the following to functions.php
      add_filter( 'wp_is_application_passwords_available', '__return_true' );

    Wordfence

    IMPORTANT: in WPStaq hosted websites that are in Dev state ( have the hosted URL as staging access website ) Wordfence plugin will need to be disabled as it will prevent the article being pushed to the website.
    • In Brute Force Protection Drop Down, disable “Prevent discovery of usernames through ‘/?author=N’ scans, the oEmbed API, the WordPress REST API, and WordPress XML Sitemaps”
    • In Brute Force Protection Drop Down, disable “Disable WordPress application passwords”
    • In Advanced Firewall Options dropdown, whitelist IPs with the following Hail IP addresses:
      • 13.213.13.58
      • 54.187.252.11
      • 18.143.159.98

    Really Simple SSL

    • Check that the setting Disable user enumeration is unticked

    Yoast SEO

    EWS Websites are hosted on WP Staq where the File Editor is not accessible. This means you will have to use FTP access to edit functions.php. Steps on doing this safely is required.
    • Add the following to functions.php
    remove_action('request', 'wordfence::preventAuthorNScans');

    There is a form submission / validation issue if “Which post type should be applied to posts for this connection?” is left blank. NONE should be manually selected in this case

    If failing to push

    • Check Upload Limit, possibly images might be too big, fix:
      • ssh into the server (ssh root@ip)
      • head to the file root (most commonly cd /var/www/html on our WP droplets)
      • open the htaccess (sudo nano .htaccess)
      • and insert after the wordpress section
      • php_value upload_max_filesize 5M
      • php_value post_max_size 7M

    (function () {
    function getBestImageUrl(img) {
    if (!img) return '';

    const srcset =
  img.getAttribute('data-srcset') ||
  img.getAttribute('data-lazy-srcset') ||
  img.getAttribute('srcset') ||
  '';

if (srcset) {
  const candidates = srcset
    .split(',')
    .map(function (item) {
      const parts = item.trim().split(/\\s+/);
      const url = parts[0];
      const width = parts[1] ? parseInt(parts[1].replace('w', ''), 10) : 0;

      return {
        url: url,
        width: isNaN(width) ? 0 : width
      };
    })
    .filter(function (item) {
      return item.url && !item.url.startsWith('data:image');
    });

  if (candidates.length) {
    candidates.sort(function (a, b) {
      return b.width - a.width;
    });

    return candidates[0].url.replace(/-\\d+x\\d+(?=\\.(jpg|jpeg|png|webp))/i, '');
  }
}

let imageUrl =
  img.getAttribute('data-full-url') ||
  img.getAttribute('data-large_image') ||
  img.getAttribute('data-src') ||
  img.getAttribute('data-lazy-src') ||
  img.getAttribute('data-eio-lazy-src') ||
  '';

if (!imageUrl) {
  const src = img.getAttribute('src') || img.currentSrc || '';

  if (src && !src.startsWith('data:image')) {
    imageUrl = src;
  }
}

return imageUrl.replace(/-\\d+x\\d+(?=\\.(jpg|jpeg|png|webp))/i, '');

    }

    function cleanDateText(text) {
    if (!text) return '';

    text = text
  .replace(/◷/g, '')
  .replace(/\\s+/g, ' ')
  .trim();

const dateMatch = text.match(/\\b(?:Jan|Feb|Mar|Apr|May|Jun|Jul|Aug|Sep|Sept|Oct|Nov|Dec)[a-z]*\\.?\\s+\\d{1,2},?\\s+\\d{4}\\b/i);

if (dateMatch) {
  return dateMatch[0].trim();
}

return text;

    }

    function getDateText(post) {
    const publishedDate = post.querySelector('.post-meta .published');

    if (publishedDate) {
  return cleanDateText(publishedDate.textContent);
}

const existingCopiedDate = post.querySelector('.hail-card-date');

if (existingCopiedDate) {
  return cleanDateText(existingCopiedDate.textContent);
}

const meta = post.querySelector('p.post-meta, .post-meta');

if (meta) {
  return cleanDateText(meta.textContent);
}

return '';

    }

    function removePagination(container) {
    const paginations = container.querySelectorAll('.pagination');

    paginations.forEach(function (pagination) {
  const wrapper = pagination.parentElement;

  if (wrapper && wrapper !== container) {
    wrapper.remove();
  } else {
    pagination.remove();
  }
});

    }

    function initHailBlogGrid() {
    const blog = document.querySelector('.et_pb_blog_0');
    if (!blog) return;

    const container = blog.querySelector('.et_pb_ajax_pagination_container');
if (!container) return;

removePagination(container);

const articles = Array.from(blog.querySelectorAll('article.et_pb_post'));

articles.forEach(function (article) {
  container.appendChild(article);
});

articles.forEach(function (post) {
  const title = post.querySelector('.entry-title');
  const titleLink = post.querySelector('.entry-title a');
  const imageLink = post.querySelector('.entry-featured-image-url');
  const img = post.querySelector('.entry-featured-image-url img, .et_pb_image_container img');
  const hailLink = post.querySelector('.post-content-inner a[href*="hail.to"]');

  if (!titleLink || !title) return;

  const imageUrl = getBestImageUrl(img);

  if (imageUrl) {
    post.style.setProperty('background-image', 'url("' + imageUrl + '")', 'important');
  }

  let textGroup = post.querySelector('.hail-card-text');

  if (!textGroup) {
    textGroup = document.createElement('div');
    textGroup.className = 'hail-card-text';
    post.appendChild(textGroup);
  }

  let dateEl = post.querySelector('.hail-card-date');
  const dateText = getDateText(post);

  if (!dateEl) {
    dateEl = document.createElement('div');
    dateEl.className = 'hail-card-date';
  }

  dateEl.textContent = dateText ? '◷ ' + dateText : '';

  if (dateText && !textGroup.contains(dateEl)) {
    textGroup.appendChild(dateEl);
  }

  if (!textGroup.contains(title)) {
    textGroup.appendChild(title);
  }

  const fallbackUrl = titleLink.href;
  const finalUrl = hailLink ? hailLink.href : fallbackUrl;

  post.dataset.finalUrl = finalUrl;
  post.dataset.opensNewTab = hailLink ? 'true' : 'false';

  titleLink.href = finalUrl;

  if (imageLink) {
    imageLink.href = finalUrl;
  }

  if (hailLink) {
    titleLink.target = '_blank';
    titleLink.rel = 'noopener';

    if (imageLink) {
      imageLink.target = '_blank';
      imageLink.rel = 'noopener';
    }
  }

  if (!post.dataset.hailGridClickReady) {
    post.dataset.hailGridClickReady = 'true';

    post.addEventListener('click', function (event) {
      const clickedLink = event.target.closest('a');

      if (clickedLink) return;

      const url = post.dataset.finalUrl;
      const opensNewTab = post.dataset.opensNewTab === 'true';

      if (!url) return;

      if (opensNewTab) {
        window.open(url, '_blank', 'noopener');
      } else {
        window.location.href = url;
      }
    });
  }
});

    }
