Setup Guide

Set up WordPress first, then connect Hail so pushed content lands in the right post categories.

WordPress

1. Create Categories

In WordPress, go to Posts -> Categories and create the categories each connection should publish into. Recommended categories include Articles, Publications, Notices, and Alerts.

Hail automatically pushes all public tags into WordPress, so extra filtering can usually be managed there.

You can also create categories based on page destinations instead of content types. For example, use Homepage, Newsletter Page, or Sports Page if that better matches the site structure.

2. Create an Application Password

Go to Users in WordPress and select the admin user that Hail should connect with.

  1. Scroll to Application Passwords.
  2. Create a new application password.

Note down these details for the Hail connection:

  • WordPress site URL
  • WordPress username for the user that owns the application password
  • WordPress application password

Hail

Create a WordPress+ Connection

  1. Go to Hail and select the organisation from the top-right menu. If you only have one organisation, it will be selected by default.
  2. Select Connections from the left sidebar.
  3. Select Blogging, then choose the WordPress Plus connection.
  4. Enter the WordPress site URL, for example https://wordpress-site.co.nz.
  5. Enter the WordPress username. Use the actual WordPress username, not the application password name.
  6. Enter the application password.
  7. Click Authorise.

After authorising, Hail will connect to the WordPress site. You can then customise the WordPress+ connection and push Hail Articles or Publications to WordPress from the standard connections panel.

If Which post type should be applied to posts for this connection? is not needed, manually select NONE to avoid the blank-field validation issue.

When editing an Article or Publication in Hail, users will see the new Push to WordPress connection in the right-hand sidebar. Clicking it sends the content to WordPress as a post using the category assigned to that connection. In WordPress, use Blog modules or post queries filtered by category to display the pushed content.

Gotchas

Wordfence

Troubleshooting: If Wordfence has disabled application passwords, enable them again by unchecking the related Wordfence setting.

If username discovery prevention blocks the connection, remove the Wordfence author scan prevention hook in functions.php:

remove_action('request', 'wordfence::preventAuthorNScans');

Network and plugin considerations

HTTP sites: if the site is not using HTTPS, add this to functions.php so application passwords are available:

add_filter('wp_is_application_passwords_available', '__return_true');

WPStaq development sites: Wordfence may need to be disabled while the site is in dev state, because it can prevent Hail from pushing articles to WordPress.

  • In Brute Force Protection, disable username discovery prevention.
  • In Brute Force Protection, disable the setting that turns off WordPress application passwords.

In Advanced Firewall Options, allowlist the Hail IP addresses:

  • 13.213.13.58
  • 54.187.252.11
  • 18.143.159.98

Really Simple SSL: make sure Disable user enumeration is unticked.

Yoast SEO / WPStaq: if the WordPress file editor is unavailable, update functions.php over FTP or another safe file-access method.

Connection form validation

If Which post type should be applied to posts for this connection? is left blank, the form can fail validation. Select NONE manually when no post type is required.

If publishing fails

Check the server upload limits. Large images can stop content from being pushed successfully.

  1. SSH into the server, for example ssh root@ip.
  2. Go to the WordPress web root, commonly /var/www/html.
  3. Open .htaccess, for example sudo nano .htaccess.
  4. Add these values after the WordPress section:
php_value upload_max_filesize 5M php_value post_max_size 7M
